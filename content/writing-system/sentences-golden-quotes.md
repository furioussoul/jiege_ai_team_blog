# 金句库（Sentences）

> 金句的作用：让人想转发、想收藏、想引用
> 原则：短、有观点、有画面感

---

## 什么是好金句？

### 标准：
1. **短**：一句话能说完
2. **有观点**：不是废话
3. **有画面感**：能想象出场景
4. **有传播性**：别人想转

---

## 金句收集

### 关于 AI 与编程

```
"AI coding agents basically didn't work before December and basically work since."
— Karpathy

// 解析：用 "basically...basically" 形成对比，制造冲击
```

```
"This is what post-AGI feels like."
— Karpathy

// 解析：短、有画面感、引发好奇
```

```
"I don't think I've typed like a line of code probably since December."
— Karpathy

// 解析：具体数字 + 反直觉，制造冲击
```

```
"I am in this perpetual state of AI psychosis."
— Karpathy

// 解析：自造词 + 夸张表达，制造记忆点
```

```
"You're not typing computer code into an editor like the way things were since computers were invented, that era is over."
— Karpathy

// 解析：长句但有力，用"era is over"制造终结感
```

### 关于工作与成长

```
"不是比谁敲得快，而是比谁'指挥'得更好。"

// 解析：对比结构，有行动指引
```

```
"我不想当那个还在敲代码的'工匠'。我要学会当那个指挥 AI 干活的'指挥官'。"

// 解析：身份对比 + 选择，有态度
```

```
"我可能再也回不去了。"

// 解析：短、有画面感、引发共鸣
```

### 关于技术与趋势

```

— Karpathy

// 解析：游戏隐喻 + 终极感
```

```

— Karpathy

// 解析：自嘲 + 时代感
```

## 金句的创作公式

### 公式1：对比式

```
不是 A，而是 B。

例子：
不是比谁敲得快，而是比谁"指挥"得更好。
```

### 公式2：转折式

```
看起来是 A，其实是 B。

例子：
看起来他"不会写"了，其实他"不用写"了。
```

### 公式3：定义式

```
X 就是 Y。

例子：
编程的本质就是"把复杂问题分解、设计解决方案、实现并验证"。
```

### 公式4：判断式

```
X 时代结束了。

例子：
从计算机发明到现在，我们都是在编辑器里敲代码。那个时代结束了。
```

### 公式5：隐喻式

```
X 就像 Y。

例子：
编程的未来，可能就像玩 StarCraft——你是指挥官，AI 是你的兵。
```

## 收集方法

### 看到好句子，问自己：

1. 它为什么打动我？
2. 它用了什么结构？
3. 我能不能仿写一个？

### 收集格式：

```
"[原句]"
— [作者]

// 解析：[它为什么好]
// 可用结构：[抽象出的公式]
// 我的仿写：[自己写一个]
```
