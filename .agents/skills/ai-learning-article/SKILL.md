# AI学习加速器文章写作协作 Skill

本skill用于协作撰写"AI提速学习"主题的文章，确保文章风格统一、要点完整。

## 触发场景

当用户要求：
- "写一篇AI学习加速器的文章"
- "按照之前的风格写AI学习文章"
- "创建AI提效学习主题的内容"

Agent应加载此skill。

---

## 文章框架

### 标题模板

```
AI学习加速器：[核心卖点]
```

示例：
- AI学习加速器：4个指令让学习效率提升10倍
- AI学习加速器：用AI重构你的知识体系

### 开篇结构

1. **真实故事引入**（必须AI相关）
   - ❌ 不要用：React、Vue、前端框架等传统技术例子
   - ✅ 要用：AI Agent、LLM、Prompt Engineering、RAG等AI相关例子
   - 故事要有反差：努力但低效 → 方法对了高效

2. **权威引用**
   - 必须引用权威人物的观点
   - 常用人物：Andrej Karpathy、Sam Altman、Richard Feynman、Joshua Tenenbaum等

3. **核心论点**
   - 一句话点明文章主旨

## 写作要点

### 1. 开头例子要求

| 要求 | 说明 |
|------|------|
| 主题相关 | 必须是AI/Agent/LLM相关，不要用传统技术 |
| 有数据 | 看了多少小时教程、做了多少Demo |
| 有反差 | 努力 but 无效 → 方法对了高效 |
| 真实感 | 面试场景、真实对话 |

**示例：**
```
❌ 简历上写着"精通React"，看了30小时教程，写了三个Todo List
✅ 简历上写着"精通AI Agent开发"，看了50小时教程，调通了LangChain的几个Demo
```

### 2. 时间对比要夸张

突出"AI提速10倍"的核心卖点，时间对比要明显：

| 传统学习 | AI辅助学习 | 提速倍数 |
|----------|------------|----------|
| 半年 | 1周 | ~25倍 |
| 一年 | 两周 | ~25倍 |
| 3个月 | 1周 | ~12倍 |

**示例：**
```
❌ "让我在两个月内从零基础到能设计架构"
✅ "让我在一周内从零基础到能设计架构"

❌ "以前看视频学了三个月还在入门阶段"
✅ "以前看视频学了半年还在入门阶段"

❌ "Rust：2周从零到能写CLI工具"
✅ "Rust：3天从零到能写CLI工具"
```

### 3. 四指令框架

文章核心是4个指令，每个指令按以下结构：

```
### 指令X：[名称]

**指令：** [具体指令模板]

[问题背景]

[权威案例]

[个人实践]

**对比：**
- 以前：[低效状态]
- 现在：[高效状态]
```

### 4. 权威案例要求

每个指令至少引用一个权威来源：

| 指令 | 常用权威来源 |
|------|-------------|
| 画学习地图 | OpenAI内部培训原则、图式理论 |
| 讲人话 | Richard Feynman费曼学习法、双重编码理论 |
| 当私教 | Google工程师培训、测试效应 |
| 做费曼 | Sam Altman、生成效应 |

### 5. 个人实践要求

- 具体技术栈（不要泛泛而谈）
- 具体场景（学习什么知识点）
- 具体效果（用了什么方法，达到什么效果）

**示例：**
```
❌ "我用这个方法学了很多东西"（太泛）
✅ "学Kubernetes时，'Pod'这个概念让我困惑了很久..."（具体）
```

### 6. 认知科学原理

文章必须包含原理支撑：

| 指令 | 认知原理 | 解决的问题 |
|------|----------|-----------|
| 画学习地图 | 图式理论（Schema） | 没有全局观，学得散 |
| 讲人话 | 双重编码（Dual Coding） | 抽象概念难以理解 |
| 当私教 | 测试效应（Testing Effect） | 没有反馈，不知是否真会 |
| 做费曼 | 生成效应（Generation Effect） | 被动接收，主动输出少 |

## 审稿检查清单

Agent审稿时，检查以下要点：

### 内容检查

- [ ] 开头例子是否AI相关？（不要React、Vue等传统技术）
- [ ] 时间对比是否足够夸张？（突出10倍提效）
- [ ] 4个指令是否完整？
- [ ] 每个指令是否有权威案例？
- [ ] 每个指令是否有个人实践？
- [ ] 是否有认知科学原理支撑？
- [ ] 结尾金句是否有力？

### 风格检查

- [ ] 是否有吸引人的开篇故事？
- [ ] 是否有清晰的结构（分段、小标题）？
- [ ] 是否有生动的比喻？
- [ ] 是否有对比表格？
- [ ] 是否有参考来源？

### 格式检查

- [ ] frontmatter是否完整（title, description, tags, image, date, author）
- [ ] 图片路径是否正确（/images/covers/xxx.png）
- [ ] 代码块是否正确标注语言
- [ ] 引用是否使用正确格式（> text）
